How to shrink a bloated WSL2 disk (ext4.vhdx)

You deleted gigabytes inside your Linux distro, but Windows still shows the space as used. WSL2 grows its virtual disk on demand and never shrinks it back on its own. Here is how to reclaim that space safely.

For WSL2 on Windows 10 and 11. Compaction only, your files are untouched.

Why WSL2 holds onto the space

Each WSL2 distribution stores its entire Linux filesystem inside a single virtual disk file called ext4.vhdx. That disk is dynamically expanding: as you install packages, pull images and build projects, the file grows to fit. The catch is that it only ever grows. When you delete files inside Linux, the space is freed inside the virtual disk, but the .vhdx file on Windows stays the same size. Docker Desktop's WSL2 backend behaves the same way and is usually the worst offender.

To give the space back to Windows, you have to compact the virtual disk: tell it to release the empty blocks it is no longer using. Compaction does not touch your data, it just shrinks the file to match what is actually inside.

Find the disk

Distro disks usually live under your local app data, for example: 

%LOCALAPPDATA%\Packages\<distro-package>\LocalState\ext4.vhdx

Docker's data disk sits under %LOCALAPPDATA%\Docker\wsl. There can be several, which is part of why this is annoying to do by hand.

The manual way with diskpart

First shut WSL down completely so nothing has the disk open: 

PS C:\> wsl --shutdown

Then compact the file with diskpart:

PS C:\> diskpart
DISKPART> select vdisk file="C:\Users\you\AppData\Local\Packages\...\ext4.vhdx"
DISKPART> attach vdisk readonly
DISKPART> compact vdisk
DISKPART> detach vdisk
DISKPART> exit
Watch out: you have to type the exact path for every disk, and run the whole sequence once per .vhdx. Miss the wsl --shutdown step and compact will fail because the disk is in use.

The one-command way with Sifty

Sifty finds every virtual disk on the system for you and compacts each one, no paths to type: 

PS C:\> sifty optimize

Pick Compact virtual hard disks from the list. Sifty discovers each .vhdx, compacts it in place, and reports how much it reclaimed. This is a non-destructive operation: it only releases empty space and never changes the contents of your Linux filesystem.

Heads up: compacting virtual disks needs an elevated terminal. Run sifty --admin optimize and Windows will prompt for elevation, or start your terminal as Administrator. sifty doctor tells you whether you are already elevated.

How often to do it

There is no need to compact constantly. Do it after a big cleanup inside Linux, after pruning Docker images (docker system prune), or whenever your C: drive starts feeling tight and WSL is the obvious culprit.
