Where the winget cache lives and how to clean it

Every time winget installs or upgrades an app, it downloads the installer to a temporary folder. Those files are not always cleaned up, and on a machine that installs a lot of software they quietly add up.

For the Windows Package Manager (winget) on Windows 10 and 11.

Where the downloads go

winget stages the installers it downloads under your local temp directory: 

%LOCALAPPDATA%\Temp\WinGet

Most of the time these are small, but large installers (think IDEs, runtimes or anything bundled) leave behind hundreds of megabytes that serve no purpose once the app is installed.

Clearing it by hand

You can open that folder and delete its contents, or do it from PowerShell: 

PS C:\> explorer $env:LOCALAPPDATA\Temp\WinGet
Why not just delete the whole folder blindly? Because a download might be in progress, and because it is easy to fat-finger the path and remove the wrong temp data. It is safer to preview what will go and send it to the Recycle Bin rather than a permanent delete.

Clearing it with Sifty

The winget download cache is one of the categories Sifty's junk scan already knows about, alongside browser caches, temp files and the rest. See it without deleting anything: 

PS C:\> sifty junk scan
  WinGet download cache   412 MB
  ...

Then clean, which previews first and asks before it acts:

PS C:\> sifty junk clean --apply

It moves the cached installers to the Recycle Bin, reports anything it had to skip because it was still in use, and logs what it removed. As with every Sifty clean, sifty undo reverses it.
